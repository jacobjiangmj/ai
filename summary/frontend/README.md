# 前端总结

## 角色定位

前端负责两件事：

- 作为 GitLab 流水线耗时看板的交互与展示层
- 提供一个独立的中国象棋本地对弈页

## 目录结构

- `src/main.js`：应用入口，根据路由路径切换页面组件
- `src/App.vue`：GitLab 看板主页面
- `src/XiangqiPage.vue`：象棋对弈页面
- `src/style.css`：全局样式与看板样式
- `vite.config.js`：Vite 配置

## 技术栈

- Vue 3 组合式 API
- Vite
- ECharts 5

## 页面入口机制

前端没有引入 Vue Router，而是用 `window.location.pathname` 做简单分发：

- `/` 对应 `App.vue`
- `/xiangqi` 对应 `XiangqiPage.vue`

这种实现很轻，适合当前只有两个页面的场景。

## 看板页总结

### 页面职责

`App.vue` 是项目当前最核心的前端文件，负责：

- 加载筛选项
- 管理筛选条件
- 保存和恢复本地偏好
- 调用同步接口
- 调用图表接口
- 初始化和销毁 ECharts 图表实例
- 渲染两种看板模式

### 已实现的用户能力

- 阶段多选
- 作业名多选
- 关键字搜索
- 快捷时间范围选择
- 自定义开始/结束时间
- 一键重置筛选
- 切换两种图表模式
- 同步 GitLab 数据
- 在本地记住用户的筛选偏好

### 核心状态

页面围绕以下状态组织：

- `dashboardType`：当前看板类型
- `filters`：筛选条件
- `filterOptions`：筛选候选项
- `charts`：图表数据
- `loading`、`syncing`、`errorMessage`：请求状态
- `openSelector`：当前打开的下拉选择器

### 数据流

1. 页面挂载时拉取 `filters` 和默认图表数据。
2. 用户修改筛选条件后，watch 自动触发重新加载图表。
3. 用户点击同步后，请求后端同步接口，再刷新筛选项和图表。
4. 图表数据更新后，重新创建 ECharts 实例并渲染。

### 图表表现

支持两种模式：

- 按日期：展示每天平均耗时
- 按作业 ID：展示每次作业耗时，并支持点击跳转 GitLab 链接

图表统一包含：

- 主折线
- 30 天平均值虚线
- tooltip 详细信息

### 交互特点

- 筛选条件使用自定义下拉面板而不是原生多选框
- 阶段和作业名支持本地搜索
- 时间范围支持快捷区间和精确到分钟的手动输入
- 筛选偏好保存在 `localStorage`

## 象棋页总结

### 页面定位

`XiangqiPage.vue` 是一个完全前端本地运行的中国象棋页面，不依赖后端。

### 已实现能力

- 标准开局布局
- 基础走法校验
- 将军判断
- 终局判定
- 最近着法记录
- 重开一局
- 基于 `vue-router` 的页面路由切换
- 人机对战、悔棋、回看与棋谱导出

### 规则覆盖

代码中已处理这些常见规则：

- 将帅九宫移动
- 士斜走
- 象不过河与塞象眼
- 马走日与别马腿
- 车直线移动
- 炮隔山打子
- 兵卒过河前后规则差异
- 将帅照面
- 禁止走出让己方主将被攻击的非法着法

### 结构特点

- 规则函数和页面状态基本写在同一组件内
- 适合单文件演示和快速迭代
- 如果后续继续扩展 AI、悔棋、联机对战，再拆分逻辑会更合理

## 样式特点

- 看板页采用偏冷色、玻璃拟态式卡片风格
- 象棋页采用暖色木纹感视觉风格
- 两个页面风格明确区分，说明前端并非完全共用一套视觉语义
- 已包含移动端适配规则

## 当前限制

- `App.vue` 体量较大，后续继续扩展时维护成本会升高
- 图表实例管理依赖手动创建和销毁
- 没有测试代码
- 看板与象棋页共享同一前端工程，但业务边界还没有进一步抽象
- 象棋规则仍未覆盖长将、长捉、重复局面和更完整和棋裁定

## 象棋页验证记录

### 本轮已确认修复

- 已引入正式前端路由，象棋页不再依赖 `pathname` 分支切换
- 象棋页与看板页之间改为路由跳转
- 着法记录和 AI 锁定提示的路数显示已按红黑双方视角区分
- 选中棋子后点击非法目标时，界面现在会给出明确提示

### 本轮发现但仍保留的风险

- 当前验证主要基于代码审查，仓库环境缺少 `node/npm` 可执行文件，未完成本机浏览器实机点测
- 棋盘虽已接近传统中国象棋样式，但仍是按钮网格驱动，不是严格意义上的交叉点建模
- 终局规则仍以基础可玩为主，未覆盖长将、长捉、重复局面等更完整裁定

## 适合后续扩展的方向

- 拆分 `App.vue` 中的筛选区、图表区、数据请求逻辑
- 给图表逻辑增加可复用组件封装
- 为象棋逻辑拆出独立规则模块
- 增加单元测试和交互测试
